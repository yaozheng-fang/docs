---
title: "Use sandboxes"
---

The `sandbox` command group creates and uses AgentKit sandbox tools. A sandbox tool can be a built-in `CodeEnv` or `SkillEnv`, or a `Private` tool backed by a custom image. Sessions are created on top of a tool and are used to execute commands, invoke an A2A agent, open a web preview, transfer files, or inject local model-login credentials.

<Warning>
Sandbox tool management requires AgentKit cloud authentication. Run `agentkit login <address>` first, or set AK/SK environment variables such as `VOLCENGINE_AGENTKIT_ACCESS_KEY` / `VOLCENGINE_AGENTKIT_SECRET_KEY` or `VOLCENGINE_ACCESS_KEY` / `VOLCENGINE_SECRET_KEY`. TOS mounting and custom image builds also require permissions for TOS and Container Registry.
</Warning>

## Tool Resolution And Local Config

Most operation commands resolve a sandbox tool before creating or using a session. Resolution checks, in order: `--tool-id` or `--tool-name`, the `AGENTKIT_SANDBOX_TOOL_ID` environment variable, the locally cached tool record, then the first Ready cloud tool matching the requested type. If no usable tool is found, commands such as `exec`, `shell`, and `web` can create one automatically when the current authentication mode supports it.

The local config file is `.agentkit/sandbox.yaml`. `sandbox create`, `sandbox build`, and automatic tool creation write common tool identifiers into this file; `sandbox config` manages defaults for models, networking, TOS, sessions, and related settings. Explicit command-line options take precedence over local config.

Common config keys:

| Config Key | Type | Description |
| --- | --- | --- |
| `region` | string | AgentKit region. |
| `tool-type` | `CodeEnv` \| `SkillEnv` \| `Private` | Default sandbox tool type. |
| `tool-id` | string | Default sandbox tool ID. |
| `tool-name` | string | Default sandbox tool name. |
| `cpu` | `2` \| `4` \| `8` \| `16` | vCPU count used when creating a tool. |
| `tos-bucket` | string | TOS bucket mounted when creating a tool. |
| `tos-mount` | string | Mount path inside the sandbox. |
| `image-url` | string | Custom image URL for `Private` tools. |
| `enable-snapshot` | boolean | Enable snapshot support when creating a tool. |
| `websearch-apikey` | string | Web search API Key injected into the sandbox. |
| `network-public` | boolean | Whether public network access is enabled. |
| `network-private` | boolean | Whether private VPC networking is enabled. |
| `network-shared-internet` | boolean | Whether shared internet egress is enabled for private networking. |
| `network-vpc-id` | string | VPC ID for private networking. |
| `network-subnet-ids` | string[] | Subnet IDs for private networking, as CSV or a JSON array. |
| `session-id` | string | Default user session ID. |
| `ttl` | number | Session TTL in seconds. |
| `git-config` | string | Source used by `exec` / `shell` when injecting Git identity. |
| `model-name` | string | Model name injected into the sandbox. |
| `model-provider` | string | Model provider. |
| `model-base-url` | string | Model API base URL. |
| `model-api-key` | string | Model API Key; `config --list` redacts it. |

## sandbox config

Set, unset, or view defaults in `.agentkit/sandbox.yaml`.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `--set <KEY=VALUE>` | Set a config value; can be repeated. | — |
| `--unset <KEY>` | Remove a config value; can be repeated. | — |
| `--list` | List the current config merged with defaults; secret fields are redacted. | `false` |

```bash

```

## sandbox create

Create a sandbox tool and wait until it reaches Ready. After creation succeeds, the CLI writes the tool ID and tool name into `.agentkit/sandbox.yaml` for later commands.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `--tool-type <type>` | Tool type: `CodeEnv`, `SkillEnv`, or `Private`. | `CodeEnv` |
| `--tool-name <name>` | Tool name; generated when omitted. | Generated |
| `--tos-bucket <bucket>` | TOS bucket to mount into the sandbox. | — |
| `--tos-mount <path>` | Mount path inside the sandbox. | `/home/gem/workspace` |
| `--cpu <count>` | vCPU count, one of `2`, `4`, `8`, `16`. Memory is 2048 MB per vCPU. | `4` |
| `--model-name <name>` | Model name injected into the sandbox. | — |
| `--model-api-key <key>` | Model API Key injected into the sandbox. | `MODEL_API_KEY` |
| `--model-provider <provider>` | Model provider. | Auto-selected by cloud provider |
| `--model-base-url <url>` | Model API base URL; non-built-in Ark URLs also require `--model-provider`. | — |
| `--websearch-apikey <key>` | Inject `WEB_SEARCH_API_KEY`. | — |
| `--image-url <url>` | Custom image URL; required for `--tool-type Private`. | — |
| `--enable-snapshot` | Enable snapshot support. | `false` |
| `--network-public` | Enable public network access. | `true` |
| `--no-network-public` | Disable public network access. | — |
| `--network-private` | Enable private VPC networking. | `false` |
| `--no-network-private` | Disable private VPC networking. | — |
| `--network-shared-internet` | Enable shared internet egress for private networking. | `false` |
| `--no-network-shared-internet` | Disable shared internet egress. | — |
| `--network-vpc-id <id>` | VPC ID for private networking. | — |
| `--network-subnet-ids <ids>` | Comma-separated subnet IDs. | — |
| `-r, --region <region>` | AgentKit region. | Auto-detect |
| `--json` | Output JSON. | `false` |

```bash
agentkit sandbox create --tool-type CodeEnv --tool-name dev-code --cpu 4 --model-api-key "$MODEL_API_KEY"
```

```bash
agentkit sandbox create --tool-type Private --tool-name custom-code --image-url cr.example.com/agentkit/sandbox:latest
```

## sandbox build

Build a custom sandbox image in cloud Code Pipeline, push it to Container Registry, and write the resulting image URL into `.agentkit/sandbox.yaml` as `image-url`. After the build completes, use `sandbox create --tool-type Private` to create a tool from that image.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `--dockerfile <path>` | Dockerfile path relative to `--project-dir`. | `Dockerfile` |
| `--image-name <name>` | Container image name, mapped to the registry repo. | `agentkit-custom-sandbox-image` |
| `--repo <name>` | Container image name; equivalent to `--image-name`. | `agentkit-custom-sandbox-image` |
| `--tag <tag>` | Container image tag; may contain `{{timestamp}}`. | `{{timestamp}}` |
| `--namespace <name>` | Container Registry namespace. | `agentkit` |
| `--project-dir <path>` | Project directory packaged as the build context. | Current directory |
| `-r, --region <region>` | AgentKit region. | `VOLCENGINE_REGION` or `cn-beijing` |

```bash
agentkit sandbox build --dockerfile Dockerfile --image-name custom-code --tag dev
agentkit sandbox create --tool-type Private --tool-name custom-code
```

Set `AGENTKIT_BUILD_TIMEOUT_MS` to adjust the cloud build wait timeout.

## sandbox exec

Open a sandbox terminal and optionally run an initial command. Use `--mode tmux` to attach to a tmux session by session ID, creating it when it does not exist.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `-s, --session-id <id>` | User session ID. | Config or generated |
| `--sid <id>` | Alias for `--session-id`. | — |
| `--tool-id <id>` | Sandbox tool ID. | Auto-resolved |
| `--tool-name <name>` | Sandbox tool name. | Auto-resolved |
| `--tool-type <type>` | Sandbox tool type. | `CodeEnv` |
| `--command <cmd>` | Initial command to run after connecting. | — |
| `--mode <mode>` | Execution mode; currently supports `tmux`. | — |
| `--shell-id <id>` | Reconnect to a remote terminal shell. | — |
| `--copy <sourceAndDestination...>` | Copy local `SOURCE` to sandbox `DESTINATION` before exec; pass source/destination pairs and repeat as needed. | — |
| `--git-config <source>` | Inject Git identity from `local` or an INI/TOML/JSON file. | Config |
| `--model-name <name>` | Model name for this session. | Config |
| `--model-api-key <key>` | Model API Key for this session. | `MODEL_API_KEY` or config |
| `--model-provider <provider>` | Model provider for this session. | Config |
| `--model-base-url <url>` | Model API base URL for this session. | Config |
| `--disable-websearch-apikey` | Clear `WEB_SEARCH_API_KEY` for this session. | `false` |
| `-r, --region <region>` | AgentKit region. | Auto-detect |

```bash
agentkit sandbox exec --session-id dev --command "codex" --mode tmux
```

```bash
agentkit sandbox exec --session-id dev --copy ./README.md sandbox:/home/gem/README.md --command "ls /home/gem"
```

## sandbox invoke

Invoke a `SkillEnv` or another A2A-compatible sandbox agent through the sandbox endpoint. By default, the command waits for the task to finish and prints JSON. With `--async`, it creates the task and returns immediately; later pass `--task-id` to poll it.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `-s, --session-id <id>` | User session ID. | Config or generated |
| `--sid <id>` | Alias for `--session-id`. | — |
| `--prompt <prompt>` | Prompt sent to the sandbox agent; required unless `--task-id` is set. | — |
| `--tool-id <id>` | Sandbox tool ID. | Auto-resolved |
| `--tool-name <name>` | Sandbox tool name. | Auto-resolved |
| `--tool-type <type>` | Sandbox tool type. | `SkillEnv` |
| `--async` | Return immediately after creating the A2A task. | `false` |
| `--task-id <id>` | Poll an existing A2A task. | — |
| `--ttl <seconds>` | Sandbox session TTL in seconds. | Config |
| `--model-name <name>` | Injected as `MODEL_AGENT_NAME`. | Env or config |
| `--model-provider <provider>` | Injected as `MODEL_AGENT_PROVIDER`. | Env or config |
| `--model-base-url <url>` | Injected as `MODEL_AGENT_API_BASE`. | Env or config |
| `--model-api-key <key>` | Injected as `MODEL_AGENT_API_KEY`. | Env or config |
| `--timeout <seconds>` | Maximum seconds to wait for task completion. | `1200` |
| `--interval <seconds>` | Poll interval in seconds. | `2` |
| `--history-length <count>` | A2A task history length to request. | `20` |
| `--a2a-path <path>` | A2A JSON-RPC path on the sandbox endpoint. | `/a2a` |
| `-r, --region <region>` | AgentKit region. | Auto-detect |

```bash
agentkit sandbox invoke --tool-type SkillEnv --session-id dev --prompt "Summarize the current project"
```

```bash
agentkit sandbox invoke --task-id task-xxxxxxxx --timeout 300
```

## sandbox run

Read `agentkit-sandbox-run.yaml` and translate its entries into multiple `sandbox exec` commands. Use it to open several sandbox terminals at once or, outside macOS, to run one command inline.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `-f, --config <path>` | Workflow YAML file. | `agentkit-sandbox-run.yaml` |
| `--terminal <count>` | Number of entries to open or run. | `1` |
| `--dry-run` | Print the generated `sandbox exec` commands without running them. | `false` |

```yaml title="agentkit-sandbox-run.yaml"
exec:
  - name: test
    session_id: dev
    cwd: .
    copy:
      - ["./README.md", "sandbox:/home/gem/README.md"]
    command: npm test
  - name: inspect
    session_id: dev
    command: ls -la /home/gem
```

```bash
agentkit sandbox run --terminal 2
agentkit sandbox run --dry-run
```

Entries may use `exec`, `execs`, `tabs`, or `commands` as the top-level key. Each entry may contain `command`, `session_id`, `tool_id`, `tool_type`, `mode`, `shell_id`, `git_config`, `model_name`, `model_api_key`, `model_provider`, `model_base_url`, `cwd` / `workdir`, and `copy` / `copies`. Use `args` or `argv` to pass raw arguments directly to `sandbox exec`.

## sandbox delete

Delete a sandbox tool, or delete a user session under a specified tool. Tool deletion requires exactly one target from `--tool-id` or `--tool-name`; pass `--session-id` to delete a session.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `--tool-id <id>` | Sandbox tool ID. | — |
| `--tool-name <name>` | Sandbox tool name; when several tools match, retry with `--tool-id`. | — |
| `-s, --session-id <id>` | User session ID to delete. | — |
| `--sid <id>` | Alias for `--session-id`. | — |
| `-r, --region <region>` | AgentKit region. | Auto-detect |
| `--force` | Skip the confirmation prompt. | `false` |

```bash
agentkit sandbox delete --tool-id tool-xxxxxxxx --force
agentkit sandbox delete --tool-id tool-xxxxxxxx --session-id dev --force
```

## sandbox codex-login

Inject local Codex or Claude OAuth login credentials into a sandbox session. This command injects subscription login credentials only; local API keys are not injected.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `-s, --session-id <id>` | User session ID to inject into. | Config or generated |
| `--sid <id>` | Alias for `--session-id`. | — |
| `-p, --provider <provider>` | Subscription to inject: `codex` or `claude`. | `codex` |
| `--auth-file <path>` | Use a specific local credential file. | — |
| `--codex-home <path>` | Local Codex home. | `$CODEX_HOME` or `~/.codex` |
| `--login` | Run `codex login` first when the local Codex credential is missing. | `true` |
| `--no-login` | Do not run `codex login` when the local Codex credential is missing. | — |
| `--tool-id <id>` | Sandbox tool ID. | Auto-resolved |
| `--tool-name <name>` | Sandbox tool name. | Auto-resolved |
| `--tool-type <type>` | Sandbox tool type. | `CodeEnv` |
| `--dry-run` | Print the redacted injection action without creating a session. | `false` |
| `-r, --region <region>` | AgentKit region. | Auto-detect |

```bash
agentkit sandbox codex-login --session-id dev --provider codex
agentkit sandbox exec --session-id dev --command "codex" --mode tmux
```

## sandbox scp

Copy one file or directory between local storage and an existing sandbox session. Exactly one side of the source/destination pair must start with `sandbox:`; relative sandbox paths are resolved under `/home/gem`.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `<source>` | Source path; sandbox paths use the `sandbox:` prefix. | Required |
| `<destination>` | Destination path; sandbox paths use the `sandbox:` prefix. | Required |
| `-s, --session-id <id>` | User session ID used for transfer, required. | Required |
| `--sid <id>` | Alias for `--session-id`. | — |
| `--tool-id <id>` | Sandbox tool ID used to disambiguate locally cached sessions. | — |

```bash
agentkit sandbox scp ./README.md sandbox:/home/gem/README.md --session-id dev
agentkit sandbox scp sandbox:/home/gem/result.json ./result.json --session-id dev
```
