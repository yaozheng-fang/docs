title: "Build & deploy"
---

The `deploy` command builds and deploys the project to AgentKit, running the full pipeline of configuration, build, and apply. It also exposes each stage as a dedicated subcommand.

## deploy

Run the full deployment pipeline (config → build → apply). The top-level options are applied during the configuration stage.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `-n, --name <name>` | Runtime/app name (config). | — |
| `-r, --region <region>` | Volcengine region (config). | — |
| `-p, --project <name>` | AgentKit project (config). | — |

```bash
agentkit deploy --name my-agent --region cn-beijing --project demo
```

## deploy config

Scaffold the `.agentkit/` directory, generating `agentkit.yaml` and a `Dockerfile`.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| `-n, --name <name>` | Runtime/app name. | — |
| `-r, --region <region>` | Volcengine region. | — |
| `-p, --project <name>` | AgentKit project. | — |
| `-f, --force` | Overwrite existing generated files. | `false` |

```bash
agentkit deploy config --name my-agent --force
```

`agentkit.yaml` is the single source of deploy configuration — both `deploy build` and `deploy apply` read from it. In the generated file, required and common fields are active; every optional field is shown commented-out with its full shape, so you uncomment and fill it to enable it. The complete file looks like this:

```yaml .agentkit/agentkit.yaml
# agentkit.yaml — AgentKit deploy configuration
# Generated by `agentkit deploy`. Active fields are uncommented; optional
# fields are shown commented-out — uncomment and fill to enable them.

# ── Project ──────────────────────────────────────────────
name: my-agent
region: cn-beijing             # cn-beijing | cn-shanghai
project: default

# ── Runtime resources ────────────────────────────────────
runtime:
  cpu_milli: 1000              # CPU in milli-cores (1000 = 1 vCPU)
  memory_mb: 2048              # memory in MB
  min_instance: 0              # 0 = scale to zero when idle
  max_instance: 5
  max_concurrency: 20          # concurrent requests per instance

# ── Build ────────────────────────────────────────────────
# Defaults to .agentkit/Dockerfile (or ./Dockerfile if present).
# dockerfile: .agentkit/Dockerfile

# ── Environment variables (injected into the runtime) ────
# Avoid committing secrets; reference them from your environment instead.
envs:
  # MODEL_AGENT_API_KEY: ""
  # LOG_LEVEL: info

# ── Model (optional) ─────────────────────────────────────
# model_agent_name: doubao-seed-1-6-250615

# ── Associated resources (optional) ──────────────────────
# knowledge_id: kb-xxxxxxxx
# memory_id: mem-xxxxxxxx
# tool_id: t-xxxxxxxx
# mcp_toolset_id: xxxxxxxx

# ── Auth (optional; pick one type) ───────────────────────
# auth:
#   type: key_auth             # key_auth | custom_jwt
#   # --- when type: key_auth ---
#   api_key_name: ""           # name of the API key (auto-created if omitted)
#   api_key_location: header   # header | query
#   # --- when type: custom_jwt ---
#   discovery_url: https://userpool-xxx.auth.id.cn-beijing.volces.com/.well-known/openid-configuration  # REQUIRED
#   allowed_clients:           # optional allow-list of client ids
#     - client-id-1

# ── Network (optional) ───────────────────────────────────
# network:
#   enable_public_network: true
#   enable_private_network: false
#   vpc:
#     vpc_id: vpc-xxxxxxxx     # REQUIRED when using a VPC
#     subnet_ids:
#       - subnet-xxxxxxxx
#     enable_shared_internet_access: false

# ── TLS / log delivery (optional) ────────────────────────
# tls:
#   enable_log: true           # REQUIRED to enable this block
#   tls_project_id: ""
#   tls_topic_id: ""

# ── Observability (optional) ─────────────────────────────
# apmplus: true                # enable APMPlus monitoring

# ── Tags (optional) ──────────────────────────────────────
# tags:
#   - { key: team, value: ai }

# ── Advanced (optional) ──────────────────────────────────
# role_name: ""                # IAM role; auto-created when omitted
# client_token: ""             # idempotency token for create

# ── Infrastructure ───────────────────────────────────────
# Where the image is built and stored. "Auto" = created & managed for you;
# replace any value with your own pre-provisioned resource if required.
infrastructure:
  container_registry:          # Volcengine Container Registry (CR)
    instance_name: Auto        # Auto → agentkit-platform-<account-id>
    namespace_name: agentkit
    repo_name: my-agent        # defaults to the app name
  tos:                         # TOS (build artifacts)
    bucket_name: Auto          # Auto → agentkit-platform-<account-id>
    object_prefix: agentkit-builds
```

Field reference:

- **Project**: `name` the runtime/app name; `region` the deploy region (`cn-beijing` or `cn-shanghai`); `project` the AgentKit project it belongs to.
- **`runtime`**: `cpu_milli` CPU in milli-cores (`1000` = 1 vCPU); `memory_mb` memory in MB; `min_instance` minimum instances (`0` scales to zero when idle); `max_instance` maximum instances; `max_concurrency` concurrent requests per instance.
- **`dockerfile`**: path to the Dockerfile used for the build, defaulting to `.agentkit/Dockerfile` (or `./Dockerfile` if one exists at the project root).
- **`envs`**: key-value pairs injected into the runtime. Don't commit secrets — reference them from your local environment instead.
- **`model_agent_name`** (optional): the model the runtime uses.
- **Associated resources** (optional): `knowledge_id`, `memory_id`, `tool_id`, and `mcp_toolset_id` attach a knowledge base, memory collection, tool, and MCP toolset respectively.
- **`auth`** (optional, pick one type): `type: key_auth` uses API-key auth with `api_key_name` (auto-created if omitted) and `api_key_location` (`header` or `query`); `type: custom_jwt` uses JWT auth, where `discovery_url` (the OIDC discovery URL) is required and `allowed_clients` is an optional client allow-list.
- **`network`** (optional): `enable_public_network` and `enable_private_network` control access; when using a VPC, `vpc.vpc_id` is required, alongside `subnet_ids` and `enable_shared_internet_access`.
- **`tls`** (optional): log delivery. Set `enable_log: true` to enable the block, then fill `tls_project_id` and `tls_topic_id`.
- **`apmplus`** (optional): set to `true` to enable APMPlus monitoring.
- **`tags`** (optional): a list of resource tags, e.g. `{ key: team, value: ai }`.
- **Advanced** (optional): `role_name` is the IAM role (auto-created when omitted); `client_token` is an idempotency token for the create call.
- **`infrastructure`**: where the image is built and stored. `Auto` means the platform creates and manages it for you; replace a value with your own resource to reuse one. `container_registry` is the Volcengine CR, with `instance_name` (`Auto` → `agentkit-platform-<account-id>`), `namespace_name`, and `repo_name` (defaults to the app name); `tos` is the object storage holding build artifacts, with `bucket_name` (`Auto` → `agentkit-platform-<account-id>`) and `object_prefix`.

## deploy build

Build the image in the cloud and write the artifact.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| _(no options)_ | This subcommand takes no options. | — |

```bash
agentkit deploy build
```

## deploy apply

Create or update the runtime from the latest built artifact.

| Flag / Argument | Description | Default |
| --- | --- | --- |
| _(no options)_ | This subcommand takes no options. | — |

```bash
agentkit deploy apply
```
