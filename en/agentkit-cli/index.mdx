`agentkit` is the command-line tool for AgentKit: scaffold projects, build and
deploy runtimes, and manage tools, knowledge bases, memory, and evaluation —
all from your terminal.

## Command groups

| Group | What it manages |
| --- | --- |
| `init` | scaffold a new agent project |
| `deploy` | build & deploy the project (`config` → `build` → `apply`) |
| `runtime` | deployed runtimes (list/show/logs/versions/release/update/delete) |
| `knowledge` (`kb`) | knowledge bases |
| `memory` (`mem`) | memory collections |
| `tool` | tools (MCP servers, code environments, …) |
| `harness` | code-free agents defined by a `harness.yaml` |
| `invoke` | call a deployed runtime |
| `auth` | SSO login/logout/whoami and login profiles |
| `onboard` | install bundled skills into local coding agents |
| `eval dataset` | evaluation datasets and their cases (`eval` prefix optional) |
| `tree` | print the command tree |

## Conventions

- `knowledge`↔`kb`, `memory`↔`mem` are aliases; the `eval` prefix is optional
  (`agentkit dataset …` == `agentkit eval dataset …`).
- Most read commands accept `--json`; region is auto-sensed unless you pass
  `-r, --region`.
- Destructive commands (`delete`, `remove`) prompt for confirmation; pass `-y`
  to skip.
