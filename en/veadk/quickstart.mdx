This tutorial gets your first VeADK agent running in a few minutes — install it,
configure a model, write a minimal script, and get a real answer back.

<Note>
  This chapter only covers "zero to first answer." Deploying to AgentKit, memory,
  knowledge bases, multi-agent, and evaluation are covered in their own chapters.
</Note>

## Prerequisites

- **Python 3.10+**
- **A Volcengine Ark API key** — create one in the [Ark console](https://console.volcengine.com/ark) to call models.

## 1. Install

```bash
pip install veadk-python
```

Optionally, install the extensions (Feishu channel, A2UI, etc.):

```bash
pip install veadk-python[extensions]
```

## 2. Configure a model

Create a `config.yaml` in your project root — VeADK reads it automatically. A
minimal agent only needs the model configured:

```yaml config.yaml
model:
  agent:
    provider: openai
    name: doubao-seed-1-6-250615
    api_base: https://ark.cn-beijing.volces.com/api/v3/
    api_key: # <-- your Volcengine Ark API key
```

<Tip>
  `provider: openai` means the OpenAI-compatible protocol; `api_base` points at
  Ark's compatible endpoint, so you can use Ark models like doubao directly.
</Tip>

## 3. Write your first agent

Create `main.py`:

```python main.py
from veadk import Agent
import asyncio

agent = Agent()

res = asyncio.run(agent.run("Introduce Volcengine in one sentence."))
print(res)
```

- `Agent` holds the model and instruction; with no arguments it uses the default
  model from `config.yaml`.
- `agent.run(...)` is async and returns the final text answer.

Run it:

```bash
python main.py
```

You'll see the model's one-sentence answer printed to your terminal. Your first
VeADK agent is now working.
