VeADK ships two frontends: **VeADK Web** for local debugging, and **VeADK Frontend**, a production-grade, enterprise agent-application frontend.

## VeADK Web and VeADK Frontend

**VeADK Web** is a visual debugging UI inherited from Google ADK Web — chat with your agent and inspect execution in the browser with zero frontend code, ideal for local development and troubleshooting; see [VeADK Web](/en/veadk/components/frontend/veadk-web).

**VeADK Frontend** is a customizable, production-grade React app — an enterprise agent workbench: create agents in the UI (conversational, custom, template, and workflow modes, generating a runnable VeADK project), chat with and debug them, browse the skill center, connect remote AgentKit agents, run full-text and web search, inspect call traces, and sign in. It talks to the same process `veadk frontend` launches, so there is no separate backend to deploy; for the command and build instructions, see [VeADK Frontend](/en/veadk/components/frontend/veadk-frontend).

## A2UI

[A2UI](/en/veadk/components/frontend/a2ui) is one of VeADK Frontend's capabilities — the rich-UI rendering layer: it lets an agent return cards, forms, and row/column layouts in a conversation instead of plain text only, rendered natively by the frontend. Enable it by setting `enable_a2ui=True` when creating the agent, and extend the built-in set with your own enterprise components.
