---
title: "使用 TOS ContextBucket 存储"
---

`tos_context` 后端由 `TosContextBucketLTMBackend` 实现，使用火山引擎 [TOS](https://www.volcengine.com/product/TOS) 的 ContextBucket 记忆能力作为长期记忆存储。它是托管服务，由服务端完成记忆推理与检索，无需自建向量库或本地 embedding。

后端把 `index`（或 `app_name`）映射为一个 **ContextBucket**，把运行时的 `user_id` 映射为该桶下的一个 **ContextSet**，从而按用户天然隔离记忆。首次使用时会自动创建对应的 ContextBucket 与 ContextSet（并启用 `memory` 场景）。

## 何时使用

- 生产环境，需要持久化与托管运维；
- 希望使用火山引擎 TOS 的 ContextBucket 记忆能力（服务端推理与检索）；
- 希望按 `user_id` 做强隔离（每个用户对应一个 ContextSet）。

## 依赖

ContextBucket API 目前只在**预发布（beta）版本**的 TOS SDK 中提供：`tos>=2.9.4b1`。VeADK 的核心依赖为稳定版 `tos>=2.8.4`（供 TOS 对象存储、VikingDB 使用），**不会**自动安装该 beta 版本。在使用 `backend="tos_context"` 前，需显式升级 TOS SDK：

```bash
pip install --upgrade "tos>=2.9.4b1" --pre
```

<Warning>
pip / uv 默认会**忽略预发布版本**，必须显式加 `--pre`（uv 使用 `--prerelease=allow`），否则不会安装上 `2.9.4b1`，运行时仍会命中下方的报错。
</Warning>

## 常见运行时报错与处理

`tos_context` 采用 fail-closed 设计：依赖或配置不满足时会在**初始化阶段直接报错**，不会静默降级或写入错误数据。常见错误及处理如下：

| 现象 | 触发条件 | 处理方式 |
| :--- | :--- | :--- |
| 提示缺少 `tos` SDK | 环境中未安装 `tos` SDK。 | 安装 SDK：`pip install --upgrade "tos>=2.9.4b1" --pre`。 |
| 提示已安装的 `tos` 版本不支持 ContextBucket | 已安装 `tos`，但版本过旧（如 `2.8.4`、`2.9.2` 稳定版），缺少 ContextBucket 相关方法。 | 升级到预发布版：`pip install --upgrade "tos>=2.9.4b1" --pre`。报错信息会附带当前已安装版本号与该命令。 |
| 提示 `account_id` 或 `control_endpoint` 缺失 | `account_id` 或 `control_endpoint` 未配置。 | 补齐环境变量 `DATABASE_TOS_CONTEXT_ACCOUNT_ID`、`DATABASE_TOS_CONTEXT_CONTROL_ENDPOINT`（见下方配置表）。 |

<Note>
ContextBucket 能力是在既有的 TOS 客户端类上**新增方法**实现的，旧版本仍可正常导入 `tos`，只是缺少这些方法。VeADK 在初始化时会探测当前 SDK 是否真正支持 ContextBucket，从而把「能导入但调用时静默失败」的隐性问题，转化为初始化阶段清晰、可操作的报错。
</Note>

## 使用示例

```python
from veadk import Agent
from veadk.memory.long_term_memory import LongTermMemory

# index 会映射为 ContextBucket 名，必须满足桶命名规则（见下方 Note）
ltm = LongTermMemory(
    backend="tos_context",
    index="demo-agent-memory",
    top_k=5,
)
agent = Agent(
    name="demo",
    long_term_memory=ltm,
    auto_save_session=True,  # 会话结束自动写入长期记忆
)
```

初始化时若 ContextBucket 不存在，VeADK 会自动创建；每个 `user_id` 首次写入或检索时，若对应的 ContextSet 不存在也会自动创建（并启用 `memory` 场景）。

也可以通过 `backend_config` 显式传入配置：

```python
from veadk.memory.long_term_memory import LongTermMemory

ltm = LongTermMemory(
    backend="tos_context",
    index="demo-agent-memory",
    backend_config={
        "account_id": "your-account-id",
        "context_bucket_name": "demo-agent-memory",
        "tos_context_config": {
            "control_endpoint": "your-control-endpoint",
            "endpoint": "tos-cn-beijing.volces.com",
            "region": "cn-beijing",
        },
    },
)
```

## 参数

### 构造参数

`TosContextBucketLTMBackend` 接受以下字段：

| 参数 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `context_bucket_name` | `str \| None` | 读取环境变量 `DATABASE_TOS_CONTEXT_BUCKET_NAME`，未设置时回退到 `index` | ContextBucket 名。 |
| `account_id` | `str \| None` | 读取环境变量 `DATABASE_TOS_CONTEXT_ACCOUNT_ID` | **必填**。账号 ID，用于 ContextBucket 控制面接口。 |
| `volcengine_access_key` | `str \| None` | 读取环境变量 `VOLCENGINE_ACCESS_KEY` | 火山引擎 Access Key（可为长期或 STS 临时 AK）。 |
| `volcengine_secret_key` | `str \| None` | 读取环境变量 `VOLCENGINE_SECRET_KEY` | 火山引擎 Secret Key（可为长期或 STS 临时 SK）。 |
| `session_token` | `str` | 读取环境变量 `VOLCENGINE_SESSION_TOKEN`，默认 `""` | STS 临时凭证的 Security Token，仅在使用临时 AK/SK 时需要。 |
| `tos_context_config` | `TOSContextBucketConfig` | 由 `TOSContextBucketConfig` 构造 | TOS 连接配置，见下表。 |

### 连接配置

`tos_context_config`（`TOSContextBucketConfig`）字段：

| 配置项 | 类型 | 默认值 | 说明 |
| :--- | :--- | :--- | :--- |
| `control_endpoint` | `str \| None` | `None` | **必填**。ContextBucket 控制面（Controller）域名。 |
| `endpoint` | `str` | `tos-cn-beijing.volces.com` | TOS 数据面域名。 |
| `region` | `str` | `cn-beijing` | 区域。 |

### 凭证获取顺序

后端优先使用显式传入或环境变量中的 `volcengine_access_key` 与 `volcengine_secret_key`（此时 STS Token 取自 `VOLCENGINE_SESSION_TOKEN`）；两者未同时提供时，回退到 VeFaaS IAM 凭证文件，从中读取临时 AK/SK 与 STS Token，适用于火山引擎云上部署（VeFaaS / AgentKit）免密运行场景。

## 环境变量配置

连接项使用环境变量前缀 `DATABASE_TOS_CONTEXT_`，本地可写在 `config.yaml` 的 `database.tos_context.*` 下（会被拍平为对应环境变量）。

| 环境变量 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `DATABASE_TOS_CONTEXT_ACCOUNT_ID` | 无 | **必填**。ContextBucket 控制面账号 ID。 |
| `DATABASE_TOS_CONTEXT_CONTROL_ENDPOINT` | 无 | **必填**。ContextBucket 控制面域名。 |
| `DATABASE_TOS_CONTEXT_BUCKET_NAME` | 回退到 `index` | ContextBucket 名。 |
| `DATABASE_TOS_CONTEXT_ENDPOINT` | `tos-cn-beijing.volces.com` | TOS 数据面域名。 |
| `DATABASE_TOS_CONTEXT_REGION` | `cn-beijing` | 区域。 |
| `VOLCENGINE_ACCESS_KEY` | 无 | 火山引擎 Access Key。 |
| `VOLCENGINE_SECRET_KEY` | 无 | 火山引擎 Secret Key。 |
| `VOLCENGINE_SESSION_TOKEN` | 无 | STS 临时凭证的 Security Token，使用长期 AK/SK 时无需设置。 |

```bash
export DATABASE_TOS_CONTEXT_ACCOUNT_ID="your-account-id"
export DATABASE_TOS_CONTEXT_CONTROL_ENDPOINT="your-control-endpoint"
export DATABASE_TOS_CONTEXT_BUCKET_NAME="demo-agent-memory"
export DATABASE_TOS_CONTEXT_ENDPOINT="tos-cn-beijing.volces.com"
export DATABASE_TOS_CONTEXT_REGION="cn-beijing"
export VOLCENGINE_ACCESS_KEY="your-ak"
export VOLCENGINE_SECRET_KEY="your-sk"
```

<Warning>
`account_id` 与 `control_endpoint` 为必填项，缺失会在初始化时报错（fail-closed，不会静默降级）。
</Warning>

<Note>
STS Token 复用全局约定 `VOLCENGINE_SESSION_TOKEN`（与 TOS 对象存储等其他组件一致），不再使用独立的 `DATABASE_TOS_CONTEXT_SECURITY_TOKEN`。使用长期 AK/SK 时无需设置。若 `VOLCENGINE_ACCESS_KEY` 与 `VOLCENGINE_SECRET_KEY` 未同时提供，会尝试读取 VeFaaS IAM 凭证文件；文件不存在时报错。
</Note>

<Note>
`index`（即 ContextBucket 名）需满足 TOS 桶命名规则：**长度 3–63**，仅包含**小写字母、数字和连字符（`-`）**，且不能以连字符开头或结尾。可用 `DATABASE_TOS_CONTEXT_BUCKET_NAME` 覆盖由 `app_name` 或 `index` 推导出的桶名。
</Note>

<Note>
记忆按 `user_id` 隔离：不同 `user_id` 写入的是不同的 ContextSet，检索时也只会命中同一 `user_id` 的记忆。跨会话召回时请确保使用一致的 `user_id`。
</Note>
