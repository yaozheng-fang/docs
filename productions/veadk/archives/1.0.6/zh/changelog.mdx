VeADK 1.0.6 发布于 2026 年 7 月 20 日。本页仅记录相对 1.0.5 的公开用户能力变化；完整提交记录见 [GitHub Release](https://github.com/volcengine/veadk-python/releases/tag/1.0.6)。

| 模块 | 变更细节 | 链接 |
| --- | --- | --- |
| Studio 部署 | `veadk studio deploy` 支持 `cn-shanghai` 部署地域（`--region` 默认仍为 `cn-beijing`），VeFaaS、API Gateway 等资源创建在所选地域；新增 `--project` 指定 VeFaaS 函数所属项目，默认为 `default`。部署时自动跨北京、上海查找 VeIdentity 用户池与客户端，跨地域命中时输出 warning 并继续。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |

VeADK 1.0.5 发布于 2026 年 7 月 19 日。本页仅记录相对 1.0.4 的公开用户能力变化；完整提交记录见 [GitHub Release](https://github.com/volcengine/veadk-python/releases/tag/1.0.5)。

| 模块 | 变更细节 | 链接 |
| --- | --- | --- |
| Studio 与 Frontend | Studio 启用与 Frontend 相同的完整界面并默认进入对话页面；新增云端 Runtime 选择、Skill Hub/本地上传/AgentKit SkillSpace、多模态会话、结构化技能与子智能体调用；重构自定义创建流程，智能模式、模板和工作流入口当前显示为「敬请期待」；并完善部署区域、环境变量、任务状态、失败重试和 OAuth Token 转发。 | [VeADK Frontend](/productions/veadk/preview/zh/components/frontend/veadk-frontend) · [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| 模型 | 默认推理模型更新为 `doubao-seed-2-1-pro-260628`；Responses API 支持回退模型与方舟上下文管理，并改进内联图片追踪。 | [模型](/productions/veadk/preview/zh/components/agent/model) |
| 运行时 | `Agent.runtime` 新增 `piagent`，支持通过 PiAgent 执行智能体并桥接函数工具、MCP 工具与技能；安装包不再内置 PiAgent 二进制文件。 | [运行时](/productions/veadk/preview/zh/components/agent/runtime) |
| Studio 测试 | 生成的智能体项目可在服务端受限临时进程中测试；测试进程默认保留 1800 秒，可通过命令参数调整。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 部署 | 使用默认 IAM Role 部署 Studio 时，自动补充访问模型、日志、追踪、知识库、记忆与身份资源所需的系统策略；显式指定的自定义 Role 不会被修改。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| AgentKit 集成 | Studio 生成的智能体应用统一通过公开函数 `create_agentkit_app` 构建，集中提供 AgentKit 接口、健康检查、拓扑、Web UI 和短期记忆配置。 | [部署到 AgentKit](/productions/veadk/preview/zh/deploy/agentkit) |
| 飞书渠道 | 改进文本、卡片和合并转发消息的内容提取，兼容飞书 Channel SDK 的同步生命周期，并支持在 Studio 生成的 AgentKit 应用中配置飞书入站消息渠道。 | [飞书渠道](/productions/veadk/preview/zh/components/extensions/feishu-channel) · [部署到 AgentKit](/productions/veadk/preview/zh/deploy/agentkit) |
