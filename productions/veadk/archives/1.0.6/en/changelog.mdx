VeADK 1.0.6 was released on July 20, 2026. This page covers public user-facing changes since 1.0.5. See the [GitHub Release](https://github.com/volcengine/veadk-python/releases/tag/1.0.6) for the complete commit history.

| Module | Change details | Link |
| --- | --- | --- |
| Studio branding | `veadk studio` and `veadk frontend` add `--site-title` (up to 6 characters, default `VeADK Studio`) and `--site-logo` (local file or HTTP(S) URL; PNG, JPEG, GIF, WebP, AVIF, ICO, ≤5 MB) to customize the system name and logo. The title sets the browser page title; the logo appears in the sidebar, login page, and favicon. The equivalent environment variables are `VEADK_SITE_TITLE` and `VEADK_SITE_LOGO`. | [Studio agent workbench](/productions/veadk/preview/en/components/frontend/studio) |
| Studio deployment | `veadk studio deploy` adds `--region` (default `cn-beijing`, also `cn-shanghai`) and `--project` (default `default`) to select the deployment region and VeFaaS function project. Deployment locates the Identity user pool and client across Beijing and Shanghai, emitting a warning and continuing on a cross-region match. Before deploying, the command checks for `ServerlessApplicationRole` and creates it with the required policies when missing. Local or remote logo images are bundled into VeFaaS at deploy time, so the deployed site does not depend on the original image URL. | [Studio agent workbench](/productions/veadk/preview/en/components/frontend/studio) |
| Studio custom create | System prompts can be authored in a rich Markdown editor with heading and list shortcuts. Root and nested Agent names are validated against Google ADK rules (start with a letter or underscore, no reserved name `user`, unique across the tree). Initialization, session, and conversation failures display credential-redacted error details and Runner logs that can be expanded and copied. | [Studio agent workbench](/productions/veadk/preview/en/components/frontend/studio) |
| Studio Runtime management | A deployed Runtime can be connected and switched to directly from the management panel; responsive layout improvements. | [Studio agent workbench](/productions/veadk/preview/en/components/frontend/studio) |
| AgentKit integration | Feishu, knowledge-base, short-/long-term memory, and observability parameters are entered in their respective feature sections, converted to VeADK runtime environment variables only at deploy time, and mirrored in the deployment environment-variable summary; they are not written to generated source or exported YAML. Toggling the Feishu channel regenerates the project automatically. Cloud image build failures display a credential-safe build-log excerpt for diagnosing dependency resolution or Dockerfile errors. | [Deploy to AgentKit](/productions/veadk/preview/en/deploy/agentkit) |

VeADK 1.0.5 was released on July 19, 2026. This page covers public user-facing changes since 1.0.4. See the [GitHub Release](https://github.com/volcengine/veadk-python/releases/tag/1.0.5) for the complete commit history.

| Module | Change details | Link |
| --- | --- | --- |
| Studio and Frontend | Studio now enables the same complete UI as Frontend and opens on the chat view by default. Added cloud Runtime selection, Skill Hub/local upload/AgentKit SkillSpace sources, multimodal conversations, and structured skill and sub-agent invocation. The custom creation flow was redesigned; intelligent, template, and workflow entries are currently marked as coming soon. Also improved deployment regions, environment variables, task status, retry behavior, and OAuth token forwarding. | [VeADK Frontend](/productions/veadk/preview/en/components/frontend/veadk-frontend) · [Studio agent workbench](/productions/veadk/preview/en/components/frontend/studio) |
| Models | Changed the default inference model to `doubao-seed-2-1-pro-260628`; Responses API now supports fallback models and Ark context management, with more reliable inline-image tracing. | [Models](/productions/veadk/preview/en/components/agent/model) |
| Runtime | Added `piagent` to `Agent.runtime`, including bridges for function tools, MCP tools, and skills; the package no longer vendors the PiAgent binary. | [Runtime](/productions/veadk/preview/en/components/agent/runtime) |
| Studio testing | Generated agent projects can run in a restricted temporary server-side test process. The process is retained for 1,800 seconds by default and can be adjusted with a command option. | [Studio agent workbench](/productions/veadk/preview/en/components/frontend/studio) |
| Studio deployment | When Studio is deployed with its default IAM role, the command adds the system policies needed to access model, logging, tracing, knowledge, memory, and identity resources. An explicitly supplied custom role is not modified. | [Studio agent workbench](/productions/veadk/preview/en/components/frontend/studio) |
| AgentKit integration | Studio-generated applications now use the public `create_agentkit_app` function to provide AgentKit endpoints, health checks, topology, the built-in Web UI, and short-term memory defaults. | [Deploy to AgentKit](/productions/veadk/preview/en/deploy/agentkit) |
| Feishu channel | Improved extraction of text, card, and merged-forward messages, added compatibility with the synchronous Channel SDK lifecycle, and allowed Studio-generated AgentKit applications to use Feishu as an inbound channel. | [Feishu channel](/productions/veadk/preview/en/components/extensions/feishu-channel) · [Deploy to AgentKit](/productions/veadk/preview/en/deploy/agentkit) |
