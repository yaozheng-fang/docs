VeADK is a Python framework for building intelligent agents. It is compatible with Google ADK's agent and runtime model and adds integrations for Ark models, AgentKit, memory, knowledge bases, observability, security, and enterprise frontends, allowing the same agent code to move from local development to cloud execution.

This documentation describes VeADK `1.0.5`.

## Use cases

- Build Python agents that call tools, search knowledge bases, or retain user memory.
- Reuse Google ADK `Agent`, `Runner`, Session, and multi-agent behavior with Volcengine services.
- Develop locally and deploy the same agent to AgentKit or VeFaaS.
- Add tracing, identity, content safety, Feishu channels, or an enterprise frontend.

## Core components

| Component | Purpose |
| --- | --- |
| `Agent` | Defines the model, instructions, tools, skills, knowledge base, memory, and execution runtime. |
| `Runner` | Executes agents and manages event streams and application, user, and session isolation. |
| Short-term sessions | Store messages and state within a conversation using memory, SQLite, MySQL, or PostgreSQL. |
| Long-term memory | Retain user facts and preferences across sessions with VikingDB, Mem0, OpenViking, and other backends. |
| Knowledge bases | Ingest and retrieve external material through managed services or vector stores. |
| Tools and skills | Connect functions, MCP services, Volcengine capabilities, and reusable skills. |
| Observability and security | Export traces and add inbound authentication, outbound credentials, and content safety. |
