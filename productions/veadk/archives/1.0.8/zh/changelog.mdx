本更新日志累计记录 VeADK 1.0.0 至 1.0.8 的全部公开用户能力变化，版本按从新到旧排列。

## 1.0.8

VeADK 1.0.8 发布于 2026 年 7 月 23 日。下表列出该版本的公开用户能力变化；完整提交记录见 [GitHub Release](https://github.com/volcengine/veadk-python/releases/tag/1.0.8)。

| 模块 | 变更细节 | 链接 |
| --- | --- | --- |
| Studio 远程 Agent | 自定义创建新增「远程 Agent」类型，仅可作为子 Agent 添加；通过 AgentKit 智能体中心 ID、召回数量、地域和 OpenAPI 地址配置，名称、描述与能力由中心返回的 Agent Card 提供；生成的内部代理在每轮请求中动态发现并挂载匹配的远程 Agent。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 部署凭据 | 部署凭据支持通过 `--volcengine-access-key` / `--volcengine-secret-key` 显式传入，或使用当前进程的 `VOLCENGINE_ACCESS_KEY` / `VOLCENGINE_SECRET_KEY` 环境变量；两者均未提供时，读取 `~/.volc/credentials` 中的 `[default]` 配置。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 部署地域 | 上海部署时 Function、API Gateway 与 AgentKit 资源保持在上海，VeFaaS Application 操作使用北京控制面端点；部署的 Function 记录所选 AgentKit 沙箱地域，使临时会话与 Skill 创建 Session 查询与 CodeEnv Tool 相同的地域。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 沙箱凭据 | Skill 创建与临时会话使用的 Ark 凭证改为用部署者 AK/SK 解析后直接写入各自的 CodeEnv Tool 作为模型环境变量，不再经过凭据托管与网关票据；浏览器与 VeFaaS Function 仅接收 Tool ID，不接触模型凭证。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 部署错误 | Tool 或模型凭证创建失败时，CLI 在脱敏凭据值后原样输出底层错误信息，便于通过服务操作、错误码、请求 ID 与请求详情定位缺失的权限或账号资源。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 对话 | 修复多轮对话中后续发送丢失历史消息的问题。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 运行时连接 | 连接 Runtime 失败时，区分 Studio 自身的归属检查与 Agent Server 兼容性、认证失败等不同原因并分别提示。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |

## 1.0.7

VeADK 1.0.7 发布于 2026 年 7 月 23 日。下表列出该版本的公开用户能力变化；完整提交记录见 [GitHub Release](https://github.com/volcengine/veadk-python/releases/tag/1.0.7)。

| 模块 | 变更细节 | 链接 |
| --- | --- | --- |
| 代码沙箱 | `run_code` 新增 `bash` 语言支持，可在 AgentKit 沙箱中执行 Shell 脚本，并提供 `exec_dir`、`env`、`hard_timeout` 与 `max_output_length` 参数，分别控制工作目录、环境变量、硬超时与输出长度上限。 | [代码沙箱](/productions/veadk/preview/zh/components/tools/code-sandbox) |
| Studio 创建模式 | 新增 Skill 创建模式：使用固定模型在两个独立的 AgentKit CodeEnv Session 中并行生成候选技能方案，完成后可对比、下载 ZIP 或发布到 AgentKit；该模式仅限 developer 与 admin 使用。同时新增临时会话模式，在 AgentKit 沙箱中启动 Codex 进行多轮对话，退出后自动清理 Session，且不写入普通历史会话。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 智能搜索 | 新增基于当前智能体的智能搜索：可在会话历史中做全文检索，或通过智能体挂载的联网搜索工具、知识库与长期记忆实时检索；Agent 元数据声明已挂载的检索源，Studio 据此提前置灰不可用的来源。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 角色与访问 | 新增基于角色的访问控制：`veadk studio`、`veadk frontend` 及 `veadk studio deploy`/`update` 支持 `--admin` 与 `--developer`（逗号分隔的用户名或 OAuth 邮箱，对应环境变量 `VEADK_STUDIO_ADMINS`、`VEADK_STUDIO_DEVELOPERS`）；登录后显示用户邮箱与角色徽标，并展示 OAuth 头像。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 更新命令 | 新增 `veadk studio update --vefaas-app-name <name>`，从本地 VeADK 源码重新构建前端并更新已有的 VeFaaS Application 与 Function，保留原 URL、SSO、IAM、网关、环境变量、标题与 Logo；未指定 `--region` 与 `--project` 时跨北京、上海及全部可见项目查找。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| Studio 部署 | `veadk studio deploy` 自动创建临时会话与 Skill 创建专用的两个 AgentKit CodeEnv Tool，也可通过 `--sandbox-chat-codex-tool-id` 与 `--sandbox-skill-creator-tool-id` 指定已有 Tool；部署后客户端登录时跳过重复的 OAuth 授权确认。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| 飞书渠道 | `FeishuChannelExtension` 支持通过飞书 OpenAPI 拉取话题历史与引用消息，新增 `include_parent_message`、`include_thread_history` 与 `thread_history_limit` 配置，并将引用或话题内容拼接到入站消息之前。 | [飞书渠道](/productions/veadk/preview/zh/components/extensions/feishu-channel) |
| Studio 工作台 | 改进自定义创建工作台：本地技能支持拖入文件夹或 ZIP 并自动识别格式，可按地域和项目浏览 AgentKit Skill Space 及其中技能；部署页展示智能体拓扑、支持导出配置与下载源码；并改进运行时管理与响应式布局、新会话启动、智能体运行时选择器、视频消息渲染与工具调用状态展示。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| 依赖 | 放宽 `psycopg2-binary` 版本约束为 `>=2.9.10`；要求 AgentKit SDK 提供默认 CORS 配置。 | [安装](/productions/veadk/preview/zh/get-started/installation) |
## 1.0.6

VeADK 1.0.6 发布于 2026 年 7 月 20 日。下表列出该版本的公开用户能力变化；完整提交记录见 [GitHub Release](https://github.com/volcengine/veadk-python/releases/tag/1.0.6)。

| Studio 部署 | `veadk studio deploy` 支持 `--region` 选择部署地域（`cn-beijing` 默认，`cn-shanghai`），VeFaaS、API Gateway 等资源创建在所选地域；新增 `--project` 指定 VeFaaS 函数所属项目，默认为 `default`。部署时自动跨北京、上海查找 VeIdentity 用户池与客户端，跨地域命中时输出 warning 并继续。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |

| Studio 创建与调试 | 系统提示词支持所见即所得的 Markdown 编辑，提供标题、列表等快捷输入；初始化、会话或对话失败时展示经过凭据脱敏的错误详情和 Runner 日志，可展开与复制。新增 Agent 名称校验，须符合 Google ADK 规则：以英文字母或下划线开头，仅含英文字母、数字和下划线，不能使用保留名 `user`，且在 Agent 结构中唯一。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| AgentKit 部署 | 飞书、知识库、短期/长期记忆和观测配置显示在各自配置区域，填写的值仅在部署时转换为运行时环境变量；选择观测 exporter 后对应 `ENABLE_*` 开关自动启用，这些值不写入生成源码或导出的 YAML，部署页同步显示便于核对。多实例部署应使用数据库持久化短期记忆；`run_sse` 返回 404 时界面提示检查会话是否落在不同实例。通过内置 Web UI 部署时，云端镜像构建失败会显示经过凭据脱敏的构建日志片段，便于定位依赖或 Dockerfile 错误。 | [部署到 AgentKit](/productions/veadk/preview/zh/deploy/agentkit) |
| Studio 运行时管理 | 改进运行时管理视图与响应式布局。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |
| 飞书渠道 | 在部署页打开或关闭飞书渠道时，Studio 自动重新生成项目，确保 `app.py`、`extensions` 依赖和运行时环境变量在部署前保持一致。 | [Studio 智能体工作台](/productions/veadk/preview/zh/components/frontend/studio) |

| 模块 | 变更细节 | 链接 |
