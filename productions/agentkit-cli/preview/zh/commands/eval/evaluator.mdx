---
title: "管理评估器"
---

`eval evaluator` 命令组用于管理评估器（负责打分的评判模型）：浏览预置模板、列出与查看已有评估器、从模板派生新评估器、删除评估器。评估器通常是一段带评分标准的提示词，配合一个大模型端点，接收若干输入字段（如 `input`、`output`、`reference_output`）后给出分数。

<Note>
所有子命令共享 `-p, --project <name>`（项目名称，默认 `default`）与 `--json`（输出原始 JSON）。
</Note>

## evaluator templates

浏览平台预置的评估器模板，作为创建自己评估器的起点。

```bash
agentkit eval evaluator templates
```

## evaluator list

列出你在当前项目下的评估器。

```bash
agentkit eval evaluator list
```

## evaluator show

查看某个评估器的详情：评分标准提示词、模型配置与输入字段。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `<id>` | 评估器 ID 或名称（必填） | 无 |

```bash
agentkit eval evaluator show 相关性
```

## evaluator create

创建一个评估器。最简单的方式是从预置模板克隆评分标准与输入 schema，再指定自己的模型端点。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--name <name>` | 评估器名称（必填） | 无 |
| `--from-template <id\|name>` | 从预置模板克隆评分标准与输入 schema | 无 |
| `--model <endpoint>` | 作为评判模型的方舟模型端点 ID（`ep-…`） | 无 |
| `--prompt-file <path>` | 自定义评分标准文本文件（覆盖模板提示词） | 无 |
| `--description <text>` | 评估器描述 | 无 |

需至少提供 `--from-template` 或 `--prompt-file` 之一。

```bash
# 从预置的「相关性」模板派生，换成自己的模型端点
agentkit eval evaluator create --name 相关性 --from-template "相关性" --model ep-xxxxxxxx
```

<Tip>
提示词中用 <code>{'{{input}}'}</code>、<code>{'{{output}}'}</code>、<code>{'{{reference_output}}'}</code> 这样的占位符引用输入字段；这些字段名即评估器的输入 schema，会在 [`eval run`](/productions/agentkit-cli/preview/zh/commands/eval/run) 中被自动映射。
</Tip>

## evaluator delete

删除一个评估器。（别名：`rm`）

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `<id>` | 评估器 ID 或名称（必填） | 无 |
| `-y, --yes` | 跳过确认提示 | `false` |

```bash
agentkit eval evaluator delete 相关性 -y
```
