---
title: "Using Harness"
tag: "NEW"
---

The harness path suits cases that need no custom code: the agent's model, tools, knowledge base, memory, and prompt all live in a single `harness.yaml`, which the CLI builds and deploys as a runtime. The whole loop revolves around that one file — initialize, configure, deploy, invoke, then return to configuration to iterate.

<Note>
  Configure credentials first. `harness deploy` is a control-plane operation and requires AK/SK; [SSO login](/productions/agentkit-cli/archives/0.46.6/en/commands/auth) currently only covers invoking (`invoke`) a deployed runtime. See the Authentication section of the [Quickstart](/productions/agentkit-cli/archives/0.46.6/en/quickstart).
</Note>

<Steps>
  <Step title="Initialize the harness">
    Create a harness directory containing `harness.yaml` and `.env.example`.

    ```bash
    agentkit harness init my-harness
    cd my-harness
    ```
  </Step>
  <Step title="Configure the agent">
    Write fields with `harness set`. Only the flags you pass are modified, so you can build up the configuration across several calls.

    ```bash
    agentkit harness set \
      --name my-harness \
      --model-name doubao-seed-1-6-250615 \
      --system-prompt "You are a concise, helpful agent." \
      --tools web_search
    ```

    <Tip>
      To configure a component such as a knowledge base or memory, set its `--<component>-type` first, then add the connection parameters; pass an empty string `""` to disable one. Running `agentkit harness set` with no flags lists the current fields. See [Harness](/productions/agentkit-cli/archives/0.46.6/en/commands/harness) for the full field list.
    </Tip>
  </Step>
  <Step title="Build and deploy">
    Build the image in the cloud from `harness.yaml` and create or update the runtime.

    ```bash
    agentkit harness deploy --project default --region cn-beijing
    ```
  </Step>
  <Step title="Invoke the runtime">
    Call the runtime you just deployed by name; the CLI resolves the endpoint and auth automatically. Invocation is a data-plane operation, so either credential type works: AK/SK, or the STS credentials from an `agentkit login` SSO session.

    ```bash
    agentkit invoke my-harness -m "Hello, introduce yourself"
    ```
  </Step>
  <Step title="Iterate">
    Change the configuration and redeploy to publish a new version; when the runtime misbehaves, read the instance logs to diagnose it.

    ```bash
    agentkit harness set --name my-harness --system-prompt "The replacement instruction."
    agentkit harness deploy
    agentkit runtime logs my-harness --limit 200
    ```
  </Step>
</Steps>

When you no longer need the runtime, delete it to free resources:

```bash
agentkit runtime delete my-harness -y
```

When you need custom logic, tools, or multi-agent orchestration, use [High-code agents](/productions/agentkit-cli/archives/0.46.6/en/workflows/high-code) instead.
