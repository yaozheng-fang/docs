---
title: "High-code agents"
---

The high-code path suits cases that need custom logic, tools, or multi-agent orchestration: you scaffold a project from a template, write the agent code directly, and let the CLI build the image in the cloud and deploy it as a runtime. The loop starts at scaffolding and runs through editing, deploying, invoking, and reading logs, then returns to editing to iterate.

<Note>
  Configure credentials first. `deploy` is a control-plane operation and requires AK/SK; [SSO login](/productions/agentkit-cli/archives/0.46.6/en/commands/auth) currently only covers invoking (`invoke`) a deployed runtime. See the Authentication section of the [Quickstart](/productions/agentkit-cli/archives/0.46.6/en/quickstart). For the full guide to writing agent code, see the [VeADK docs](/productions/veadk/preview/en/index).
</Note>

<Steps>
  <Step title="Scaffold a project">
    Create a project from a built-in template, grouped by language. Run `agentkit init -l` to list every language and template first.

    ```bash
    agentkit init my-agent -L python -t basic-agent
    cd my-agent
    ```
  </Step>
  <Step title="Edit the agent">
    Modify the scaffolded code to adjust the agent's instruction, tools, and orchestration logic.

    <Note>
      The only hard deploy requirement is that the container listens on `0.0.0.0:8000`, the runtime's readiness probe. Templates satisfy this by default, so you rarely need to change it.
    </Note>
  </Step>
  <Step title="Build and deploy">
    Run the full deploy flow (config → build → apply), building the image in the cloud and creating or updating the runtime.

    ```bash
    agentkit deploy --name my-agent --region cn-beijing
    ```

    <Tip>
      To inspect or hand-edit `agentkit.yaml` or the `Dockerfile` between scaffolding and building, run `agentkit deploy config`, `agentkit deploy build`, and `agentkit deploy apply` separately. See [Build & deploy](/productions/agentkit-cli/archives/0.46.6/en/commands/deploy).
    </Tip>
  </Step>
  <Step title="Invoke the runtime">
    Call the runtime by name to verify the deployment.

    ```bash
    agentkit invoke my-agent -m "Hello, introduce yourself"
    ```
  </Step>
  <Step title="Inspect and iterate">
    Read the logs when the runtime misbehaves; edit the code and redeploy to publish a new version.

    ```bash
    agentkit runtime logs my-agent --limit 200
    agentkit deploy
    ```

    To roll back or adjust resources, use the version and update commands: list history with `agentkit runtime versions my-agent`, publish a specific version with `agentkit runtime release my-agent --rev <n>`, and change CPU and memory with `agentkit runtime update my-agent --cpu-milli 1000 --memory-mb 2048 --auto-release`.
  </Step>
</Steps>

When you no longer need the runtime, delete it to free resources:

```bash
agentkit runtime delete my-agent -y
```

When you only need configuration and no code, use [Using Harness](/productions/agentkit-cli/archives/0.46.6/en/workflows/harness) instead.
