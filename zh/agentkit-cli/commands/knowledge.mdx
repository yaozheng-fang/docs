---
title: "管理知识库"
---

`knowledge` 命令组用于管理 AgentKit 知识库，涵盖列表、详情查看、创建、更新与删除等操作。该命令同时提供 `kb` 与 `knowledgebase` 别名。

## knowledge list

列出所有知识库。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-r, --region <region>` | Volcengine 区域。 | 自动检测 |
| `-p, --project <name>` | 项目名称。 | — |
| `--json` | 输出原始 JSON。 | `false` |

```bash
agentkit knowledge list --project demo
```

## knowledge show

查看知识库的详情。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `<id>` | 知识库 id，例如 `kb-xxxxxxxx`。必填。 | — |
| `-r, --region <region>` | Volcengine 区域。 | 自动检测 |
| `--json` | 输出原始 JSON。 | `false` |

```bash
agentkit knowledge show kb-12345678
```

## knowledge create

创建知识库。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-n, --name <name>` | 知识库名称。 | — |
| `--provider-knowledge-id <id>` | 待注册的提供方侧知识库 id。 | — |
| `--provider-type <type>` | 提供方类型，例如 `viking`。 | — |
| `-d, --description <text>` | 知识库描述。 | — |
| `-p, --project <name>` | 项目名称。 | — |
| `-r, --region <region>` | Volcengine 区域。 | 来自环境变量/默认值 |
| `--json <jsonString>` | 高级：合并进请求体的 JSON 对象（覆盖各选项）。 | — |

```bash
agentkit knowledge create --name docs-kb --provider-type viking --provider-knowledge-id pk-123
```

## knowledge update

更新知识库。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `<id>` | 知识库 id，例如 `kb-xxxxxxxx`。必填。 | — |
| `-d, --description <text>` | 新的描述。 | — |
| `--vpc-id <id>` | 要挂载的 VPC id。 | — |
| `--subnet-ids <ids>` | 逗号分隔的子网 id（需配合 `--vpc-id`）。 | — |
| `-r, --region <region>` | Volcengine 区域。 | 自动检测 |
| `--json <jsonString>` | 高级：合并进请求体的 JSON 对象（覆盖各选项）。 | — |

```bash
agentkit knowledge update kb-12345678 --description "更新后的文档库"
```

## knowledge delete

删除知识库。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `<id>` | 知识库 id，例如 `kb-xxxxxxxx`。必填。 | — |
| `-r, --region <region>` | Volcengine 区域。 | 自动检测 |
| `-y, --yes` | 跳过确认提示。 | `false` |

```bash
agentkit knowledge delete kb-12345678 --yes
```
