---
title: "使用沙箱"
---

`sandbox` 命令组用于创建和使用 AgentKit 沙箱工具。沙箱工具可以是内置的 `CodeEnv`、`SkillEnv`，也可以是自定义镜像对应的 `Private` 工具；会话建立在工具之上，用于执行命令、调用 A2A 智能体、打开 Web 预览、传输文件或注入本地模型登录凭据。

<Warning>
管理沙箱工具需要完成 AgentKit 云端鉴权。可先运行 `agentkit login <address>`，或按部署环境设置 `VOLCENGINE_AGENTKIT_ACCESS_KEY` / `VOLCENGINE_AGENTKIT_SECRET_KEY`、`VOLCENGINE_ACCESS_KEY` / `VOLCENGINE_SECRET_KEY` 等 AK/SK 环境变量。涉及 TOS 挂载或自定义镜像构建时，还需要可访问 TOS 与容器镜像仓库的权限。
</Warning>

## 工具解析与本地配置

多数操作命令会解析一个沙箱工具。解析顺序为：命令行的 `--tool-id` 或 `--tool-name`、环境变量 `AGENTKIT_SANDBOX_TOOL_ID`、本地缓存的工具记录、云端第一个处于 Ready 状态且类型匹配的工具。若仍未找到可用工具，`exec`、`shell`、`web` 等命令会在支持的鉴权方式下按当前配置自动创建工具并等待就绪。

本地配置文件位于 `.agentkit/sandbox.yaml`。`sandbox create`、`sandbox build` 与自动创建工具时会写入常用的工具标识；`sandbox config` 可集中设置模型、网络、TOS、会话等默认值。命令行显式传入的选项优先级高于本地配置。

常用配置键：

| 配置键 | 类型 | 说明 |
| --- | --- | --- |
| `region` | string | AgentKit 区域。 |
| `tool-type` | `CodeEnv` \| `SkillEnv` \| `Private` | 默认沙箱工具类型。 |
| `tool-id` | string | 默认沙箱工具 ID。 |
| `tool-name` | string | 默认沙箱工具名称。 |
| `cpu` | `2` \| `4` \| `8` \| `16` | 创建工具时的 vCPU 数。 |
| `tos-bucket` | string | 创建工具时挂载的 TOS 桶名称。 |
| `tos-mount` | string | TOS 在沙箱内的挂载路径。 |
| `image-url` | string | `Private` 工具使用的自定义镜像地址。 |
| `enable-snapshot` | boolean | 创建工具时启用快照能力。 |
| `websearch-apikey` | string | 注入到沙箱的 Web 搜索 API Key。 |
| `network-public` | boolean | 是否启用公网访问。 |
| `network-private` | boolean | 是否启用私有 VPC 网络。 |
| `network-shared-internet` | boolean | 私有网络下是否启用共享公网出口。 |
| `network-vpc-id` | string | 私有网络使用的 VPC ID。 |
| `network-subnet-ids` | string[] | 私有网络使用的子网 ID，可用逗号分隔或 JSON 数组。 |
| `session-id` | string | 默认用户会话 ID。 |
| `ttl` | number | 会话 TTL，单位为秒。 |
| `git-config` | string | `exec` / `shell` 注入 Git 身份时使用的来源。 |
| `model-name` | string | 注入沙箱的模型名称。 |
| `model-provider` | string | 模型提供方。 |
| `model-base-url` | string | 模型 API Base URL。 |
| `model-api-key` | string | 模型 API Key，`config --list` 会脱敏显示。 |

## sandbox init

初始化自定义沙箱镜像构建文件，在当前目录创建 `Dockerfile`。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-f, --force` | 覆盖已存在的 `Dockerfile`。 | `false` |

```bash
agentkit sandbox init
```

## sandbox config

设置、删除或查看 `.agentkit/sandbox.yaml` 中的沙箱默认配置。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--set <KEY=VALUE>` | 设置配置项，可重复传入。 | — |
| `--unset <KEY>` | 删除配置项，可重复传入。 | — |
| `--list` | 列出合并默认值后的当前配置，密钥字段会脱敏。 | `false` |

```bash
agentkit sandbox config --set tool-type=CodeEnv --set cpu=4 --set model-provider=model_square
agentkit sandbox config --set model-api-key="$MODEL_API_KEY"
agentkit sandbox config --list
```

## sandbox create

创建一个沙箱工具并等待其达到 Ready 状态。创建成功后，CLI 会将工具 ID 与工具名称写入 `.agentkit/sandbox.yaml`，供后续命令复用。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--tool-type <type>` | 工具类型：`CodeEnv`、`SkillEnv` 或 `Private`。 | `CodeEnv` |
| `--tool-name <name>` | 工具名称；省略时自动生成。 | 自动生成 |
| `--tos-bucket <bucket>` | 挂载到沙箱的 TOS 桶。 | — |
| `--tos-mount <path>` | TOS 在沙箱内的挂载路径。 | `/home/gem/workspace` |
| `--cpu <count>` | vCPU 数，只能为 `2`、`4`、`8`、`16`。内存按每 vCPU 2048 MB 配置。 | `4` |
| `--model-name <name>` | 注入沙箱的模型名称。 | — |
| `--model-api-key <key>` | 注入沙箱的模型 API Key。 | `MODEL_API_KEY` |
| `--model-provider <provider>` | 模型提供方。 | 按云厂商自动选择 |
| `--model-base-url <url>` | 模型 API Base URL；非内置方舟地址需同时指定 `--model-provider`。 | — |
| `--websearch-apikey <key>` | 注入 `WEB_SEARCH_API_KEY`。 | — |
| `--image-url <url>` | 自定义镜像地址；`--tool-type Private` 时必填。 | — |
| `--enable-snapshot` | 启用快照能力。 | `false` |
| `--network-public` | 启用公网访问。 | `true` |
| `--no-network-public` | 禁用公网访问。 | — |
| `--network-private` | 启用私有 VPC 网络。 | `false` |
| `--no-network-private` | 禁用私有 VPC 网络。 | — |
| `--network-shared-internet` | 在私有网络下启用共享公网出口。 | `false` |
| `--no-network-shared-internet` | 禁用共享公网出口。 | — |
| `--network-vpc-id <id>` | 私有网络使用的 VPC ID。 | — |
| `--network-subnet-ids <ids>` | 逗号分隔的子网 ID。 | — |
| `-r, --region <region>` | AgentKit 区域。 | 自动探测 |
| `--json` | 输出 JSON。 | `false` |

```bash
agentkit sandbox create --tool-type CodeEnv --tool-name dev-code --cpu 4 --model-api-key "$MODEL_API_KEY"
```

```bash
agentkit sandbox create --tool-type Private --tool-name custom-code --image-url cr.example.com/agentkit/sandbox:latest
```

## sandbox build

在云端 Code Pipeline 中构建自定义沙箱镜像，将镜像推送到容器镜像仓库，并把生成的镜像地址写入 `.agentkit/sandbox.yaml` 的 `image-url`。构建完成后可用 `sandbox create --tool-type Private` 创建使用该镜像的工具。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--dockerfile <path>` | Dockerfile 路径，相对于 `--project-dir`。 | `Dockerfile` |
| `--image-name <name>` | 容器镜像名称，对应镜像仓库 repo。 | `agentkit-custom-sandbox-image` |
| `--repo <name>` | 容器镜像名称，与 `--image-name` 等价。 | `agentkit-custom-sandbox-image` |
| `--tag <tag>` | 镜像标签；可使用 `{{timestamp}}`。 | `{{timestamp}}` |
| `--namespace <name>` | 容器镜像仓库命名空间。 | `agentkit` |
| `--project-dir <path>` | 打包为构建上下文的项目目录。 | 当前目录 |
| `-r, --region <region>` | AgentKit 区域。 | `VOLCENGINE_REGION` 或 `cn-beijing` |

```bash
agentkit sandbox build --dockerfile Dockerfile --image-name custom-code --tag dev
agentkit sandbox create --tool-type Private --tool-name custom-code
```

可用 `AGENTKIT_BUILD_TIMEOUT_MS` 调整云端构建等待超时时间。

## sandbox exec

打开沙箱终端并可选执行初始命令，适合交互式任务。若传入 `--mode tmux`，CLI 会附加到同名 tmux 会话，不存在时创建。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-s, --session-id <id>` | 用户会话 ID。 | 配置或自动生成 |
| `--sid <id>` | `--session-id` 的别名。 | — |
| `--tool-id <id>` | 沙箱工具 ID。 | 自动解析 |
| `--tool-name <name>` | 沙箱工具名称。 | 自动解析 |
| `--tool-type <type>` | 沙箱工具类型。 | `CodeEnv` |
| `--command <cmd>` | 连接终端后执行的初始命令。 | — |
| `--mode <mode>` | 执行模式；当前支持 `tmux`。 | — |
| `--shell-id <id>` | 重新连接远端终端 shell。 | — |
| `--copy <sourceAndDestination...>` | 执行前复制本地 `SOURCE` 到沙箱 `DESTINATION`，以成对参数传入，可重复。 | — |
| `--git-config <source>` | 注入 Git 身份，值为 `local` 或 INI/TOML/JSON 文件路径。 | 配置 |
| `--model-name <name>` | 本次会话使用的模型名称。 | 配置 |
| `--model-api-key <key>` | 本次会话使用的模型 API Key。 | `MODEL_API_KEY` 或配置 |
| `--model-provider <provider>` | 本次会话使用的模型提供方。 | 配置 |
| `--model-base-url <url>` | 本次会话使用的模型 API Base URL。 | 配置 |
| `--disable-websearch-apikey` | 将本次会话的 `WEB_SEARCH_API_KEY` 置空。 | `false` |
| `-r, --region <region>` | AgentKit 区域。 | 自动探测 |

```bash
agentkit sandbox exec --session-id dev --command "codex" --mode tmux
```

```bash
agentkit sandbox exec --session-id dev --copy ./README.md sandbox:/home/gem/README.md --command "ls /home/gem"
```

## sandbox invoke

通过沙箱端点上的 A2A 接口调用 `SkillEnv` 或兼容 A2A 的沙箱智能体。默认会等待任务结束并输出 JSON；传入 `--async` 时只创建任务并立即返回，可之后用 `--task-id` 轮询。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-s, --session-id <id>` | 用户会话 ID。 | 配置或自动生成 |
| `--sid <id>` | `--session-id` 的别名。 | — |
| `--prompt <prompt>` | 发送给沙箱智能体的提示词；未传 `--task-id` 时必填。 | — |
| `--tool-id <id>` | 沙箱工具 ID。 | 自动解析 |
| `--tool-name <name>` | 沙箱工具名称。 | 自动解析 |
| `--tool-type <type>` | 沙箱工具类型。 | `SkillEnv` |
| `--async` | 创建 A2A 任务后立即返回。 | `false` |
| `--task-id <id>` | 轮询已有 A2A 任务。 | — |
| `--ttl <seconds>` | 沙箱会话 TTL，单位为秒。 | 配置 |
| `--model-name <name>` | 注入为 `MODEL_AGENT_NAME`。 | 环境或配置 |
| `--model-provider <provider>` | 注入为 `MODEL_AGENT_PROVIDER`。 | 环境或配置 |
| `--model-base-url <url>` | 注入为 `MODEL_AGENT_API_BASE`。 | 环境或配置 |
| `--model-api-key <key>` | 注入为 `MODEL_AGENT_API_KEY`。 | 环境或配置 |
| `--timeout <seconds>` | 等待任务完成的最长秒数。 | `1200` |
| `--interval <seconds>` | 轮询间隔秒数。 | `2` |
| `--history-length <count>` | 请求的 A2A 任务历史长度。 | `20` |
| `--a2a-path <path>` | 沙箱端点上的 A2A JSON-RPC 路径。 | `/a2a` |
| `-r, --region <region>` | AgentKit 区域。 | 自动探测 |

```bash
agentkit sandbox invoke --tool-type SkillEnv --session-id dev --prompt "总结当前项目"
```

```bash
agentkit sandbox invoke --task-id task-xxxxxxxx --timeout 300
```

## sandbox run

读取 `agentkit-sandbox-run.yaml`，把其中的条目转换为多条 `sandbox exec` 命令。该命令适合一次打开多个沙箱终端或在非 macOS 环境中顺序执行单个命令。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-f, --config <path>` | 工作流 YAML 文件。 | `agentkit-sandbox-run.yaml` |
| `--terminal <count>` | 要打开或执行的条目数量。 | `1` |
| `--dry-run` | 只打印将执行的 `sandbox exec` 命令。 | `false` |

```yaml title="agentkit-sandbox-run.yaml"
exec:
  - name: test
    session_id: dev
    cwd: .
    copy:
      - ["./README.md", "sandbox:/home/gem/README.md"]
    command: npm test
  - name: inspect
    session_id: dev
    command: ls -la /home/gem
```

```bash
agentkit sandbox run --terminal 2
agentkit sandbox run --dry-run
```

条目可使用 `exec`、`execs`、`tabs` 或 `commands` 作为顶层键；每个条目可写 `command`、`session_id`、`tool_id`、`tool_type`、`mode`、`shell_id`、`git_config`、`model_name`、`model_api_key`、`model_provider`、`model_base_url`、`cwd` / `workdir`、`copy` / `copies`。也可用 `args` 或 `argv` 直接写入传给 `sandbox exec` 的原始参数。

## sandbox delete

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `--tool-id <id>` | 沙箱工具 ID。 | — |
| `--tool-name <name>` | 沙箱工具名称；若匹配多个工具会要求改用 `--tool-id`。 | — |
| `-s, --session-id <id>` | 要删除的用户会话 ID。 | — |
| `--sid <id>` | `--session-id` 的别名。 | — |
| `-r, --region <region>` | AgentKit 区域。 | 自动探测 |
| `--force` | 跳过确认提示。 | `false` |

```bash
agentkit sandbox delete --tool-id tool-xxxxxxxx --force
agentkit sandbox delete --tool-id tool-xxxxxxxx --session-id dev --force
```

## sandbox codex-login

## sandbox scp

在本地与已有沙箱会话之间复制单个文件或目录。源路径和目标路径必须恰好有一侧以 `sandbox:` 开头；相对的沙箱路径会解析到 `/home/gem` 下。

| 标志/参数 | 说明 | 默认值 |

| `<destination>` | 目标路径；沙箱路径使用 `sandbox:` 前缀。 | 无（必填） |
| `-s, --session-id <id>` | 用于传输的用户会话 ID，必填。 | 无（必填） |
| `--sid <id>` | `--session-id` 的别名。 | — |
| `--tool-id <id>` | 用于区分本地缓存会话的沙箱工具 ID。 | — |

agentkit sandbox scp ./README.md sandbox:/home/gem/README.md --session-id dev
agentkit sandbox scp sandbox:/home/gem/result.json ./result.json --session-id dev
