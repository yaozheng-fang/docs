---
title: "构建与部署"
---

`deploy` 命令用于构建并将项目部署到 AgentKit，依次执行配置、构建与应用的完整流程。同时，该流程的各个阶段也以独立子命令的形式提供。

## deploy

执行完整的部署流程（config → build → apply）。顶层选项将在配置阶段生效。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-n, --name <name>` | 运行时/应用名称（配置阶段）。 | — |
| `-r, --region <region>` | Volcengine 区域（配置阶段）。 | — |
| `-p, --project <name>` | AgentKit 项目（配置阶段）。 | — |

```bash
agentkit deploy --name my-agent --region cn-beijing --project demo
```

## deploy config

脚手架生成 `.agentkit/` 目录，其中包含 `agentkit.yaml` 与 `Dockerfile`。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| `-n, --name <name>` | 运行时/应用名称。 | — |
| `-r, --region <region>` | Volcengine 区域。 | — |
| `-p, --project <name>` | AgentKit 项目。 | — |
| `-f, --force` | 覆盖已生成的文件。 | `false` |

```bash
agentkit deploy config --name my-agent --force
```

`agentkit.yaml` 是部署配置的唯一来源，`deploy build` 与 `deploy apply` 都从中读取。生成的文件里，必填与常用字段处于启用状态，其余可选字段以注释形式给出完整结构——取消注释并填值即可启用。完整内容如下：

```yaml .agentkit/agentkit.yaml
# agentkit.yaml — AgentKit deploy configuration
# Generated by `agentkit deploy`. Active fields are uncommented; optional
# fields are shown commented-out — uncomment and fill to enable them.

# ── Project ──────────────────────────────────────────────
name: my-agent
region: cn-beijing             # cn-beijing | cn-shanghai
project: default

# ── Runtime resources ────────────────────────────────────
runtime:
  cpu_milli: 1000              # CPU in milli-cores (1000 = 1 vCPU)
  memory_mb: 2048              # memory in MB
  min_instance: 0              # 0 = scale to zero when idle
  max_instance: 5
  max_concurrency: 20          # concurrent requests per instance

# ── Build ────────────────────────────────────────────────
# Defaults to .agentkit/Dockerfile (or ./Dockerfile if present).
# dockerfile: .agentkit/Dockerfile

# ── Environment variables (injected into the runtime) ────
# Avoid committing secrets; reference them from your environment instead.
envs:
  # MODEL_AGENT_API_KEY: ""
  # LOG_LEVEL: info

# ── Model (optional) ─────────────────────────────────────
# model_agent_name: doubao-seed-1-6-250615

# ── Associated resources (optional) ──────────────────────
# knowledge_id: kb-xxxxxxxx
# memory_id: mem-xxxxxxxx
# tool_id: t-xxxxxxxx
# mcp_toolset_id: xxxxxxxx

# ── Auth (optional; pick one type) ───────────────────────
# auth:
#   type: key_auth             # key_auth | custom_jwt
#   # --- when type: key_auth ---
#   api_key_name: ""           # name of the API key (auto-created if omitted)
#   api_key_location: header   # header | query
#   # --- when type: custom_jwt ---
#   discovery_url: https://userpool-xxx.auth.id.cn-beijing.volces.com/.well-known/openid-configuration  # REQUIRED
#   allowed_clients:           # optional allow-list of client ids
#     - client-id-1

# ── Network (optional) ───────────────────────────────────
# network:
#   enable_public_network: true
#   enable_private_network: false
#   vpc:
#     vpc_id: vpc-xxxxxxxx     # REQUIRED when using a VPC
#     subnet_ids:
#       - subnet-xxxxxxxx
#     enable_shared_internet_access: false

# ── TLS / log delivery (optional) ────────────────────────
# tls:
#   enable_log: true           # REQUIRED to enable this block
#   tls_project_id: ""
#   tls_topic_id: ""

# ── Observability (optional) ─────────────────────────────
# apmplus: true                # enable APMPlus monitoring

# ── Tags (optional) ──────────────────────────────────────
# tags:
#   - { key: team, value: ai }

# ── Advanced (optional) ──────────────────────────────────
# role_name: ""                # IAM role; auto-created when omitted
# client_token: ""             # idempotency token for create

# ── Infrastructure ───────────────────────────────────────
# Where the image is built and stored. "Auto" = created & managed for you;
# replace any value with your own pre-provisioned resource if required.
infrastructure:
  container_registry:          # Volcengine Container Registry (CR)
    instance_name: Auto        # Auto → agentkit-platform-<account-id>
    namespace_name: agentkit
    repo_name: my-agent        # defaults to the app name
  tos:                         # TOS (build artifacts)
    bucket_name: Auto          # Auto → agentkit-platform-<account-id>
    object_prefix: agentkit-builds
```

字段说明：

- **项目**：`name` 运行时/应用名称；`region` 部署区域（`cn-beijing` 或 `cn-shanghai`）；`project` 所属 AgentKit 项目。
- **`runtime`（运行时资源）**：`cpu_milli` CPU 毫核（`1000` = 1 vCPU）；`memory_mb` 内存 MB；`min_instance` 最小实例数（`0` 表示空闲时缩容到零）；`max_instance` 最大实例数；`max_concurrency` 单实例并发请求数。
- **`dockerfile`**：构建镜像所用的 Dockerfile 路径，默认 `.agentkit/Dockerfile`；若项目根目录存在 `./Dockerfile` 则改用它。
- **`envs`（环境变量）**：注入运行时的键值对。请勿提交密钥，改为从本地环境引用。
- **`model_agent_name`（可选）**：运行时使用的模型名称。
- **关联资源（可选）**：`knowledge_id`、`memory_id`、`tool_id`、`mcp_toolset_id` 分别关联知识库、记忆集合、工具与 MCP 工具集。
- **`auth`（可选，二选一）**：`type: key_auth` 为 API Key 鉴权，配 `api_key_name`（省略时自动创建）与 `api_key_location`（`header` 或 `query`）；`type: custom_jwt` 为 JWT 鉴权，`discovery_url`（OIDC discovery 地址）必填，`allowed_clients` 为可选的客户端白名单。
- **`network`（可选）**：`enable_public_network`、`enable_private_network` 控制公/私网访问；使用 VPC 时 `vpc.vpc_id` 必填，并配 `subnet_ids` 与 `enable_shared_internet_access`。
- **`tls`（可选）**：日志投递。启用需 `enable_log: true`，并填 `tls_project_id`、`tls_topic_id`。
- **`apmplus`（可选）**：设为 `true` 启用 APMPlus 监控。
- **`tags`（可选）**：资源标签列表，形如 `{ key: team, value: ai }`。
- **高级（可选）**：`role_name` 为 IAM 角色（省略时自动创建）；`client_token` 为创建操作的幂等 token。
- **`infrastructure`（基础设施）**：镜像的构建与存储位置。`Auto` 表示由平台自动创建并托管，如需复用已有资源可替换为自己的值。`container_registry` 为容器镜像仓库 CR，含 `instance_name`（`Auto` → `agentkit-platform-<account-id>`）、`namespace_name`、`repo_name`（默认应用名）；`tos` 为存放构建产物的对象存储，含 `bucket_name`（`Auto` → `agentkit-platform-<account-id>`）与 `object_prefix`。

## deploy build

在云端构建镜像并写入构建产物。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| _（无选项）_ | 该子命令不接受任何选项。 | — |

```bash
agentkit deploy build
```

## deploy apply

基于最新构建产物创建或更新运行时。

| 标志/参数 | 说明 | 默认值 |
| --- | --- | --- |
| _（无选项）_ | 该子命令不接受任何选项。 | — |

```bash
agentkit deploy apply
```
