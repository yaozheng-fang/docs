---
title: "使用 Harness"
tag: "NEW"
---

harness 路径适合无需自定义代码的场景：智能体的模型、工具、知识库、记忆与提示词全部写在一个 `harness.yaml` 里，由 CLI 直接构建并部署为运行时。整个闭环只围绕一个文件展开——初始化、配置、部署、调用，再回到配置持续迭代。

<Note>
  开始前请先完成[鉴权与登录](/zh/agentkit-cli/commands/auth)。CLI 会对每一次 Volcengine OpenAPI 请求签名，缺少凭证时命令会失败。
</Note>

<Steps>
  <Step title="初始化 harness">
    创建一个 harness 目录，其中包含 `harness.yaml` 与 `.env.example`。

    ```bash
    agentkit harness init my-harness
    cd my-harness
    ```
  </Step>
  <Step title="配置智能体">
    用 `harness set` 逐项写入字段。仅显式传入的标志会被修改，因此可以分多次补充配置。

    ```bash
    agentkit harness set \
      --name my-harness \
      --model-name doubao-seed-1-6-250615 \
      --system-prompt "你是一个简洁、乐于助人的智能体。" \
      --tools web_search
    ```

    <Tip>
      配置知识库、记忆等组件时，先设置其 `--<组件>-type`，再补充连接参数；传入空字符串 `""` 可禁用某个组件。不带任何标志运行 `agentkit harness set` 会列出当前字段。完整字段见[Harness](/zh/agentkit-cli/commands/harness)。
    </Tip>
  </Step>
  <Step title="构建并部署">
    根据 `harness.yaml` 在云端构建镜像，并创建或更新运行时。

    ```bash
    agentkit harness deploy --project default --region cn-beijing
    ```
  </Step>
  <Step title="调用运行时">
    按名称调用刚部署的运行时，CLI 会自动解析端点与鉴权。

    ```bash
    agentkit invoke my-harness -m "你好，介绍一下你自己"
    ```
  </Step>
  <Step title="迭代">
    修改配置后重新部署即可发布新版本；运行时行为异常时，查看实例日志定位问题。

    ```bash
    agentkit harness set --name my-harness --system-prompt "更换后的指令。"
    agentkit harness deploy
    agentkit runtime logs my-harness --limit 200
    ```
  </Step>
</Steps>

不再需要该运行时时，将其删除以释放资源：

```bash
agentkit runtime delete my-harness -y
```

需要自定义逻辑、工具或多智能体编排时，改用[高代码智能体](/zh/agentkit-cli/workflows/high-code)。
